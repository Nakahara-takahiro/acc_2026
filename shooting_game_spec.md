## 3. 画面仕様

### 3.1 表示方向

SSD1306を**縦長（ポートレート）**で使用する。

- 物理解像度：128×64 px
- 論理解像度：**64×128 px**（90°回転して使用）
  - X軸：0〜63（横方向）
  - Y軸：0〜127（縦方向、上が0・下が127）

### 3.2 画面レイアウト

```
┌────────────────────┐ Y=0
│  スコア表示エリア  │ Y=0〜9
├────────────────────┤ Y=10
│                    │
│   ゲームフィールド  │
│   (敵機が降下)      │
│                    │
├────────────────────┤ Y=118
│  自機エリア        │ Y=118〜127
└────────────────────┘ Y=127
 X=0              X=63
```

## 4. キャラクター仕様

### 4.1 自機

| 項目 | 内容 |
|------|------|
| 形状 | ▲（三角形、上向き） |
| サイズ | 幅9px、高さ7px |
| 初期位置 | X=中央（X=28）、Y=固定（Y=120） |
| 移動方向 | X軸のみ（左右） |
| 移動範囲 | X=0〜55（画面端でクランプ） |
| 移動量 | ロータリーエンコーダー1パルス = 2px |

```
  *        ← 頂点 (cx, cy-6)
 ***
*****      ← 底辺 (cx-4〜cx+4, cy)
```

### 4.2 敵機

| 項目 | 内容 |
|------|------|
| 形状 | ◆（菱形） |
| サイズ | 幅9px、高さ9px |
| 出現位置Y | Y=10（ゲームフィールド上端） |
| 出現位置X | ランダム（X=4〜59） |
| 移動方向 | Y軸のみ（下向き） |
| 移動速度 | 全敵機共通（定数 `ENEMY_SPEED = 1` px/フレーム） |
| 同時最大数 | 5体 |
| 出現間隔 | 60フレームごとに1体追加 |

```
  *        ← 上頂点
 ***
*****      ← 中央 (cx-4〜cx+4)
 ***
  *        ← 下頂点
```

### 4.3 弾

| 項目 | 内容 |
|------|------|
| 形状 | 縦2px × 横2px の矩形 |
| 発射位置 | 自機頂点から上（自機X中央, Y=自機Y-7） |
| 移動方向 | Y軸のみ（上向き） |
| 移動速度 | `BULLET_SPEED = 4` px/フレーム |
| 同時最大数 | 3発 |
| 消滅条件 | 画面上端到達（Y<0）、または敵機との衝突 |

## 5. ゲームロジック

### 5.1 状態遷移

```
電源ON
  │
  ▼
[INIT] ─────────────────► [PLAYING]
                               │
                      敵機が下端到達
                               │
                               ▼
                          [GAME OVER]
                               │
                      タクトスイッチ押下
                               │
                               └──────► [PLAYING] (リセット)
```

### 5.2 ゲームオーバー条件

- 敵機のY座標が `Y >= 118`（自機エリア上端）に到達したとき

### 5.3 衝突判定

弾と敵機の矩形が重なった場合に衝突とみなす。

- 弾のAABB：`(bx, by)〜(bx+1, by+1)`
- 敵機のAABB：`(ex-4, ey-4)〜(ex+4, ey+4)`
- 条件：`bx <= ex+4 and bx+1 >= ex-4 and by <= ey+4 and by+1 >= ey-4`

### 5.4 スコア

| イベント | 加点 |
|----------|------|
| 敵機撃墜 | +10点 |

- スコアは画面上部（Y=1）に `SCORE:XXXX` 形式で表示
- ゲームオーバー時にスコアを保持表示

## 6. 入力仕様

### 6.1 ロータリーエンコーダー

- A相（GP17）の立下りエッジ割り込みで検出
- 割り込み時にB相（GP16）を読み取り回転方向を判定
  - B相 = 1（HIGH）→ 時計回り → 自機を右へ移動
  - B相 = 0（LOW）→ 反時計回り → 自機を左へ移動
- **チャタリング防止**：前回割り込みから `5ms` 未満の割り込みは無視（`ticks_ms()` で管理）

### 6.2 タクトスイッチ（GP15）

- **チャタリング防止**：メインループ内でポーリング。前回状態変化から `50ms` 未満の変化は無視
- アクティブLow（押下時 = 0）
- 押下検出（立下り）で弾発射 or ゲームオーバー画面からの再スタート

---

## 7. ゲームループ

```
メインループ（約20fps / フレーム周期 50ms）
  │
  ├─ 入力処理
  │   ├─ エンコーダー割り込みによる自機位置更新（非同期）
  │   └─ スイッチポーリング → 弾発射 or リスタート
  │
  ├─ 更新処理
  │   ├─ 弾の移動（Y -= BULLET_SPEED）
  │   ├─ 敵機の移動（Y += ENEMY_SPEED）
  │   ├─ 敵機の出現（フレームカウンタで管理）
  │   ├─ 衝突判定（弾×敵機）
  │   └─ ゲームオーバー判定（敵機Y >= 118）
  │
  └─ 描画処理
      ├─ 画面クリア
      ├─ スコア描画
      ├─ 自機描画
      ├─ 弾描画
      ├─ 敵機描画
      └─ display.show()
```

## 8. 定数一覧

```python
# 画面
SCREEN_W       = 64    # 論理幅
SCREEN_H       = 128   # 論理高
FIELD_TOP      = 10    # ゲームフィールド上端Y
PLAYER_Y       = 120   # 自機Y座標（固定）
PLAYER_INIT_X  = 28    # 自機初期X座標
PLAYER_STEP    = 2     # エンコーダー1パルスの移動量(px)

# 弾
BULLET_SPEED   = 4     # 弾の移動速度(px/frame)
MAX_BULLETS    = 3     # 同時発射数上限

# 敵機
ENEMY_SPEED    = 1     # 敵機移動速度(px/frame)
MAX_ENEMIES    = 5     # 同時出現数上限
ENEMY_INTERVAL = 60    # 出現間隔(frame)

# 入力
ENC_DEBOUNCE_MS  = 5   # エンコーダーチャタリング防止(ms)
SW_DEBOUNCE_MS   = 50  # スイッチチャタリング防止(ms)

# フレームレート
FRAME_MS       = 50    # 1フレーム時間(ms) ≒ 20fps
                       # ※ SSD1306 I2C 400kHz転送に約25〜30ms要するため
                       #   30fps(33ms)はshow()だけで予算超過。20fpsが現実的。
                       # ※ SW_DEBOUNCE_MS=50msとフレーム周期が一致し挙動が安定。
```
